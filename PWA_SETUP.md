# Collective Dream Observatory PWA Setup

The app is now configured as an installable Progressive Web App through `vite-plugin-pwa`.

## App Identity

- App name: `Collective Dream Observatory`
- Short name: `Dream Observatory`
- Chinese name used in UI copy: `集體夢境觀測站`
- Spanish name used in UI copy: `Observatorio Colectivo de Sueños`

## What Is Cached

The service worker precaches the app shell generated by Vite:

- `index.html`
- built JavaScript and CSS assets
- local icons and static UI assets

Runtime caching is limited to same-origin static assets such as `/icons/*`, `/app-icon.svg`, and `/assets/*`.

## What Is Not Cached

Private data is intentionally not added to public Cache Storage:

- Firebase Auth responses
- Firestore private `Records`
- private dream text
- private dream images and sketches
- raw diary import files
- user exports
- research CSV/JSON export payloads
- account/profile data

Offline dream drafts are stored privately in IndexedDB on the user's device, not in Cache Storage.

## Offline Draft Flow

On the dream recording page:

1. If the user goes offline, the current draft can be saved locally.
2. If the user submits while offline, the draft is saved with `waiting_for_connection`.
3. The Offline Drafts panel lets the user restore, upload, delete, or clear local drafts.
4. A draft is not marked uploaded until the normal Firebase `createDreamRecord` path succeeds.
5. Drafts are not public until uploaded and confirmed through the normal sharing choice.

## Install Instructions

Supported browsers may show the in-app `Install Dream Observatory` prompt.

iOS Safari:

1. Open the site in Safari.
2. Tap Share.
3. Tap Add to Home Screen.

Desktop browsers:

Use the browser install button in the address bar or the browser menu.

## Update Flow

When a new service worker is available, the app shows an update prompt. If the recorder is open, it dispatches a local save event before updating so current dream text can be kept as an offline draft.

## Verification

Run:

```bash
npm run icons:pwa
npm run test:pwa
npm run build
npm run preview
```

In Chrome DevTools:

1. Open Application.
2. Confirm the manifest is valid.
3. Confirm the service worker is registered in production preview.
4. Test offline refresh loads the app shell.
5. Confirm Cache Storage does not contain private Firebase/Supabase dream data.
